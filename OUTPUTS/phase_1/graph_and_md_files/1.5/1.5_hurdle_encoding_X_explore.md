## 1. Στρατηγική: Hurdle Encoding
Για τη βέλτιστη διαχείριση της διττής φύσης των δεδομένων μικροβιώματος (παρουσία/απουσία vs σχετική ποσότητα), εγκαταλείφθηκε ο απλός CLR μετασχηματισμός με pseudocounts και υιοθετήθηκε το **Hurdle Encoding**. 

Εφαρμόστηκε "Uniform treatment", δηλαδή χωρίς κάποιο ειδικό sample split. Για κάθε ένα από τα **107 species** που επιβίωσαν από τα φίλτρα, δημιουργήθηκαν 2 νέα features ανά δείγμα:

1. **Δείκτης Παρουσίας/Απουσίας (PA - Presence/Absence):** 
   - `PA_i = 1` αν $x_i > 0$
   - `PA_i = 0` αν $x_i = 0$
2. **Σχετική Ποσότητα (rCLR - robust Centered Log-Ratio):**
   - `rCLR_i = log(x_i / geomean)` (όταν $x_i > 0$)
   - `rCLR_i = 0` (όταν $x_i = 0$)

> [!IMPORTANT]  
> Ο γεωμετρικός μέσος (geomean) υπολογίστηκε ανά δείγμα **αυστηρά** μόνο από τα μη-μηδενικά (non-zero) species αυτού του δείγματος, **χωρίς** τη χρήση pseudocount.

## 2. Δημιουργία Πίνακα `X_explore` (Βήμα 2.0)

Ολόκληρο το filtered dataset πέρασε από τη Hurdle κωδικοποίηση για τις ανάγκες της Φάσης 2 (Exploratory Data Analysis / EDA).

| Metric | Details |
|---|---|
| **Input Data** | `species_filtered_crossstudy.csv` |
| **Input Features** | 107 βακτηριακά είδη |
| **Αριθμός Δειγμάτων**| 1,604 ασθενείς / controls |
| **Τελικό Output** | `X_explore.csv` |
| **Output Features** | **214** (107 PA + 107 rCLR) |
| **Sparsity Handling**| Structural zeros vs Volumetric variance |

**Τοποθεσία του Export:**  
`data/crc_study_final_data/species_level/X_explore.csv`

Το αρχείο `X_explore` είναι πλέον έτοιμο. Σκοπός του **δεν** είναι να τροφοδοτήσει απ' ευθείας το Machine Learning pipeline (το οποίο θα τρέξει PA/rCLR μετασχηματισμούς εσωτερικά στα LOSO splits), αλλά να χρησιμεύσει ως solid βάση για unsupervised μεθόδους (Clustering, PCA, t-SNE) προκειμένου να κατανοήσουμε τη δομή των δεδομένων σε ολόκληρο το δείγμα.
